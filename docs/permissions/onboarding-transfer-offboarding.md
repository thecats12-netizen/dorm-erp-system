# 입사 · 부서이동 · 퇴사 처리 절차

대상: 인사·총무 담당자, 계정 정리 담당자

---

## 1. 입사(신규 계정) 처리

화면: **시스템 > 사용자관리 > 계정 등록**

1. 사용자 계정 생성(아이디·이름·비밀번호)
2. 이메일 인증 확인(해당 시)
3. 소속·기본정보 확인
4. **기본 역할 선택**(관리자/뷰어/기숙사 관리자/하자접수 담당자)
5. 사용자 정의 권한(추가 권한) 필요 여부 확인
6. (필요 시) 계정 저장 후 다시 열어 **추가 권한 배정 + 데이터 범위 설정**
7. **최종 권한 미리보기** 확인
8. 승인자 확인
9. 저장
10. 사용자에게 로그인 안내
11. 사용자 본인 로그인 후 **실제 메뉴 접근 확인**
12. **감사로그** 기록 확인

체크박스:
- [ ] 기본 역할이 업무에 맞다
- [ ] 추가 권한은 최소 범위이다
- [ ] 임시/외부 인력이면 종료일을 정했다
- [ ] 최종 권한 미리보기를 확인했다
- [ ] 로그인·메뉴 접근을 확인했다

---

## 2. 부서 이동 · 업무 변경 처리

> 핵심: **기존 권한을 그대로 두고 새 권한만 얹지 않는다.** 먼저 회수, 그다음 재설정.

1. **불필요한 기존 추가 권한 먼저 회수**(선택 해제 후 저장)
2. 새 역할/새 추가 권한 배정
3. **데이터 범위 재설정**(지역·기숙사·공정·담당자)
4. **다운로드 권한 재검토**
5. 적용 방식(추가/선택) 재검토
6. **최종 권한 미리보기** 확인
7. 변경 후 **재로그인 또는 새로고침**으로 반영 확인
8. **감사로그** 확인

체크박스:
- [ ] 이전 업무 권한을 회수했다
- [ ] 새 범위가 과도하지 않다
- [ ] 미리보기로 최종 권한을 확인했다

---

## 3. 퇴사 · 계약 종료 처리

> 원칙: **퇴사 즉시 접근 차단**. 데이터·감사 기록은 보존.

1. **계정 사용중지**(비활성화) — 로그인 차단
2. 활성 세션 종료(재로그인 시 권한 없음 확인)
3. **사용자 정의 권한(추가 권한) 회수**
4. 기본 역할 최소화 또는 계정 비활성화 유지
5. 다운로드 권한 제거(추가 권한 회수로 자동 정리)
6. 파일·데이터 소유권 확인(담당 데이터 인수인계)
7. 담당 데이터 인계자 지정
8. **감사로그 보존**(삭제하지 않음)
9. 계정 삭제 여부 결정(아래 4항)
10. 개인정보 보존 정책 확인(보존 기간·마스킹)

체크박스:
- [ ] 계정 비활성화 완료
- [ ] 추가 권한 회수 완료
- [ ] 담당 데이터 인수인계 완료
- [ ] 감사로그 보존 확인

---

## 4. 사용중지 vs 삭제 구분

| 처리 | 언제 | 결과 |
|---|---|---|
| **사용중지(비활성화)** | 퇴사 직후, 휴직 | 로그인 차단, 데이터·이력 보존, 복구 쉬움 |
| **Soft Delete** | 보존 기간 내 정리 | 목록에서 제외, 복구·감사 가능 |
| **완전 삭제** | 보존 기간 경과·법적 요건 충족 후 | 되돌릴 수 없음. **개인정보 보존 정책 확인 후에만** |

> ⚠️ 마지막 **관리자(admin)** 계정은 비활성화/강등이 시스템적으로 차단된다. 퇴사 전 **다른 관리자**를 먼저 지정한다.
