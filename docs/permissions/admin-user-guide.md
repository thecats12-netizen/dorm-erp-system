# 관리자 사용자 매뉴얼

대상: 권한을 직접 설정하는 관리자
전제: **시스템 > 권한관리** 및 **시스템 > 사용자관리** 는 관리자(admin)만 접근 가능하다.

> 공통 주의: 권한 변경은 대상 사용자의 **다음 로그인 또는 새로고침 후** 적용된다.

---

## A. 사용자 정의 권한(Custom Role) 생성 절차

화면: **시스템 > 권한관리 > [+ 신규 권한]**

1. 시스템 > 권한관리 진입
2. **[+ 신규 권한]** 클릭
3. **권한명** 입력 (예: `평택 남성 기숙사 조회자`)
4. **권한 코드** 입력 (영문 소문자·숫자·밑줄, 예: `pyeongtaek_male_viewer`) — 중복·시스템 기본 코드 불가
5. **설명** 입력 (누가·왜 쓰는 권한인지)
6. **권한 적용 방식** 선택
   - 기존 권한에 추가(additive) / 선택한 메뉴만 허용(restrictive)
7. 저장 후 다시 **수정** 화면을 열어 **메뉴·기능 권한** 선택
   - 전체 선택 / 모듈 전체 / 메뉴별 전체 / 읽기·쓰기·다운로드·승인 일괄 / 검색 / 접기·펼치기 지원
8. **데이터 범위** 설정(지역/성별/기숙사/시험 공정/본인 등록)
9. **최종 확인** — 위험 권한(삭제/승인/개인정보 등) 포함 여부 확인
10. **저장**
11. **테스트 사용자에게 임시 배정**
12. 테스트 계정으로 **실제 동작 확인**(메뉴·버튼·데이터 범위)
13. 이상 없으면 **운영 사용자에게 배정**

> ⚠️ **신규 권한을 만들자마자 다수 사용자에게 배정하지 않는다.** 반드시 1명 테스트 후 확대한다.

체크박스:
- [ ] 권한명·코드·설명 작성
- [ ] 적용 방식 선택(추가/선택)
- [ ] 메뉴·기능 최소 권한으로 선택
- [ ] 데이터 범위 설정
- [ ] 테스트 계정 검증 완료
- [ ] 운영 배정

---

## B. 권한 수정 절차 (영향 분석 필수)

기존 권한 수정은 **배정된 모든 사용자에게 영향**을 준다.

수정 전 반드시 확인:
- [ ] 현재 배정 사용자 수(상세보기의 "적용 사용자 수")
- [ ] 영향받는 부서
- [ ] 메뉴/기능/데이터 범위/다운로드 변경 내용
- [ ] 선택(restrictive) 사용자 영향
- [ ] 하자접수 담당자 영향 없음(이 권한은 하자 계정에 배정되지 않음)
- [ ] 수정 전/후 비교(변경 요약 확인)
- [ ] 승인 여부
- [ ] 감사로그 확인
- [ ] 롤백 방법(변경 전 상태를 기록 또는 다시 되돌리는 방법)

> ⚠️ **다수 사용자에게 배정된 권한 수정 = 고위험 작업.** 저트래픽 시간대에, 승인 후 진행한다.

---

## C. 권한 복제 가이드

화면: 권한 목록의 각 행 **[복제]**

- 시스템 기본 권한/사용자 정의 권한 모두 복제 가능(원본은 절대 변경되지 않음).
- 복제 시 **새 권한 코드 필수**, 권한명 수정 가능.
- **원본의 적용 방식(추가/선택)과 메뉴·기능 권한이 함께 복제**된다. 데이터 범위·이름·코드는 재검토한다.
- **원본에 배정된 사용자는 복제되지 않는다**(빈 상태로 생성).

주의:
- [ ] 이름·코드 변경
- [ ] 적용 방식 확인
- [ ] 메뉴·기능·데이터 범위 재검토
- [ ] 테스트 후 운영 적용
> 복제본을 원본과 동일하다고 신뢰하지 않는다. 반드시 재검토한다.

---

## D. 사용중지 / Soft Delete / 복구

| 동작 | 의미 | 데이터 |
|---|---|---|
| **사용중지** | 권한을 잠시 비활성. 신규 적용 차단 | 보존 |
| **삭제(Soft Delete)** | 일반 목록에서 제외. 물리 삭제 아님 | 보존(복구·감사 가능) |
| **복구** | 삭제된 권한 되살림 | 기존 메뉴·기능·범위 재확인 필요 |

- **사용 중인 권한(배정 계정 있음)은 즉시 삭제 불가** — 먼저 계정에서 회수 후 삭제.
- **시스템 기본 권한은 수정·삭제하지 않는다**(복제·상세보기만).
- 복구 후에는 메뉴·기능·데이터 범위가 의도대로인지 재확인한다.

---

## E. 사용자에게 권한 배정 절차

화면: **시스템 > 사용자관리 > 계정 클릭(수정) > "추가 권한" 영역**

1. 사용자 계정 수정 화면 진입
2. **추가 권한** 영역에서 부여할 사용자 정의 권한 **다중 선택**(검색 지원)
3. **최종 권한 미리보기** 확인
   - 보이는 메뉴 / 기능 / **데이터 범위**(지역·성별·기숙사·공정·본인) / 적용 방식
4. 변경 요약(추가/해제) 확인 → **추가 권한 저장**
5. 안내: **다음 로그인/새로고침 후 적용**

주의:
- 하자접수 담당자·기숙사 관리자 계정에는 추가 권한이 부여되지 않는다(보호).
- 중복 배정은 시스템이 방지한다.
- 회수: 선택 해제 후 저장(기존 배정은 Soft Delete 처리).
