# 권한 신청 · 승인 · 회수 절차

## 1. 승인 흐름
```
신청자 → 부서 관리자 → 시스템 관리자(admin) → 보안/책임자 → 적용 → 사후 확인
```
- 소규모 조직은 "부서 관리자 = 승인자, 시스템 관리자 = 적용자"로 최소 2인 검토를 유지한다.
- **신청자와 최종 승인자는 가능하면 다른 사람**으로 한다(업무 분리 원칙).

## 2. 권한 신청서 필수 항목
| 항목 | 예시 |
|---|---|
| 사용자 | 홍길동 / hong@company.com |
| 소속 | 총무팀 |
| 요청 역할(또는 Custom Role) | 평택 남성 기숙사 조회자 |
| 요청 메뉴 | 기숙사관리 > 입주자 |
| 요청 기능 | 조회, 상세 |
| 요청 데이터 범위 | 지역: 평택 / 성별: 남성 |
| 다운로드 필요 여부 | Excel 필요 / 불필요 |
| 신청 사유 | 평택 남성 기숙사 현황 집계 |
| 적용 시작일 / 종료일 | 2026-08-01 / 2026-12-31 |
| 승인자 | 총무팀장 |
| 비고 | — |

## 3. 반려 기준 (권한 과다 요청)
아래에 해당하면 **반려하거나 범위를 축소**한다.
- [ ] 업무와 무관한 메뉴/기능 요청
- [ ] "전체 지역·전체 기숙사·전체 공정"을 사유 없이 요청
- [ ] 조회만 필요한데 등록·수정·삭제·승인 요청
- [ ] 외부/단기 인력인데 종료일 미기재
- [ ] 다운로드(Excel/PDF) 필요성 불명확
- [ ] 개인정보 열람이 업무상 불필요
- [ ] 관리자(admin) 권한을 정당한 사유 없이 요청

## 4. 회수 절차
- 업무 종료·부서 이동·퇴사 시 **추가 권한을 먼저 회수**한다(사용자관리 > 추가 권한에서 선택 해제 후 저장).
- 임시 권한은 **종료일 도래 시 회수** 여부를 월간 점검에서 확인한다.
- 회수 후 **감사로그**에 변경 기록이 남았는지 확인한다.

## 체크박스
- [ ] 신청서에 시작일/종료일이 기재되어 있다
- [ ] 요청 범위가 업무 최소 범위이다
- [ ] 승인자가 신청자와 다르다
- [ ] 적용 후 테스트 계정/본인으로 동작을 확인했다
