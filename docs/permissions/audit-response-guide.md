# 감사 대응 자료 추출 절차

대상: 보안·감사 담당자
목적: 내부/외부 감사 요청 시 필요한 권한 자료를 신속·정확하게 준비한다.

## 준비할 자료 목록
| 자료 | 추출 위치(화면/데이터) | 비고 |
|---|---|---|
| 현재 사용자 목록 | 시스템 > 사용자관리 | 활성/비활성 포함 |
| 역할별 사용자 목록 | 사용자관리(역할 컬럼) | 기본 역할별 |
| Custom Role 목록 | 시스템 > 권한관리 (내보내기) | 사용중/사용중지/삭제 상태 |
| 사용자별 최종 권한 | 사용자관리 > 계정 > 최종 권한 미리보기 | 메뉴·기능·범위 |
| 데이터 범위 설정 | 권한관리 > 권한 상세(범위 미리보기) | 지역/성별/기숙사/공정/본인 |
| 최근 권한 변경 이력 | 감사로그(custom_role_audit_logs) | 변경자·전/후·시각 |
| 권한 신청·승인 기록 | 조직 문서(신청서 보관함) | 앱 외부 관리 |
| 퇴사자 권한 회수 기록 | 감사로그 + 계정 상태 | 비활성·회수 확인 |
| 관리자 접근 이력 | 보안 감사로그(security_audit_logs) + Supabase Auth 로그 | 로그인·권한 변경 |
| 다운로드 이력 | (조직 정책상 로그 위치) | 앱 내 별도 다운로드 로그가 없으면 서버/네트워크 로그 |
| 거부된 접근 이력 | Supabase 로그(RLS 42501/PGRST) + security_audit_logs | 정상 차단 vs 오류 구분 |

## 추출 절차
1. 감사 범위·기간 확정(대상 사용자/모듈/기간)
2. 위 표에서 해당 자료 식별
3. 화면 내보내기(권한관리 목록 CSV) + 감사로그 조회
4. **개인정보 마스킹** 적용(아래)
5. 자료 목록·추출자·추출일 기록
6. 책임자 검토 후 제출

## 개인정보 마스킹 원칙
- 이메일: `ho***@company.com`
- 이름: 필요 시 성만 표기(예: `홍**`)
- 연락처/식별번호: 부분 마스킹
- **access token·비밀번호·원문 개인정보는 절대 포함하지 않는다**(감사로그에도 저장되지 않음).
- 감사 목적에 필요한 최소 항목만 추출한다.

## 주의
- 자료 추출 자체가 개인정보 처리이므로, **추출·전달 이력**을 남긴다.
- 외부 제출 시 전달 경로·보관 기간을 명시한다.
