# 신규 관리자 교육자료 (60분)

대상: 신규 관리자
방식: 60분 강의 + 실습. 각 항목에 [교육 목표 / 핵심 설명 / 실습 / 주의사항 / 확인 질문].

---

## 1. 권한 시스템 개요 (5분)
- 목표: 권한이 "메뉴 접근 + 기능 + 데이터 범위" 3층으로 구성됨을 이해.
- 핵심: 기본 역할(고정 4종) + 사용자 정의 권한(관리자가 생성). 변경은 다음 로그인/새로고침 후 적용.
- 실습: 시스템 > 권한관리, 사용자관리 화면 위치 확인.
- 주의: 시스템 기본 권한은 수정·삭제 불가(복제만).
- 확인 질문: 권한의 3개 층은?

## 2. 기본 역할과 Custom Role (10분)
- 목표: 4개 기본 역할과 Custom Role의 차이 이해.
- 핵심: 관리자/뷰어/기숙사 관리자/하자접수 담당자. 업무 추가는 Custom Role로.
- 실습: 권한관리 목록에서 기본 역할(🔒)과 사용자 정의 권한 구분.
- 주의: **하자접수 담당자는 청소·하자 2메뉴만**. 추가 권한 부여 불가.
- 확인 질문: 하자접수 담당자에게 시험관리 권한을 줄 수 있는가? (답: 불가)

## 3. 적용 방식 additive/restrictive (10분)
- 목표: "기존 권한에 추가" vs "선택한 메뉴만 허용" 구분.
- 핵심: 추가=기본 유지+더하기(넓어짐), 선택=선택만(나머지 숨김).
- 실습: 각 방식으로 권한 1개씩 만들고 미리보기 비교.
- 주의: 선택(restrictive)은 필수 메뉴 누락 시 업무 불가.
- 확인 질문: 외부 협력업체에는 어느 방식? (답: 선택/restrictive)

## 4. 메뉴·기능 권한 설정 (10분)
- 목표: 메뉴 트리에서 최소 기능만 부여.
- 핵심: 전체/모듈/메뉴별 선택, 읽기·쓰기·다운로드·승인 일괄, 검색.
- 실습: "입주자 조회+상세만" 권한 만들기(등록·수정·삭제·Excel 제외).
- 주의: 다운로드/삭제/승인은 별도 신중히.
- 확인 질문: 조회만 필요한 사용자에게 삭제를 줘도 되는가? (답: 아니오)

## 5. 데이터 범위 설정 (10분)
- 목표: 지역/성별/기숙사/공정/본인 범위와 합집합·교집합 이해.
- 핵심: 같은 유형=합집합, 다른 유형=교집합, 전체 선택 시 다른 조건 자동 해제.
- 실습: "평택 + 남성" 범위 만들고 미리보기 확인.
- 주의: 범위 미설정 + 선택(restrictive) = 0건.
- 확인 질문: "서울+여성"의 의미는? (답: 서울이면서 여성)

## 6. 사용자 권한 배정 (5분)
- 목표: 계정에 Custom Role 배정 + 최종 미리보기 확인.
- 핵심: 사용자관리 > 추가 권한(다중 선택) → 미리보기 → 저장.
- 실습: 테스트 계정에 권한 배정 후 미리보기 확인.
- 주의: 배정은 테스트 후 확대. 다음 로그인 후 적용.
- 확인 질문: 권한 변경이 즉시 반영되지 않는 이유는?

## 7. 입사·이동·퇴사 처리 (5분)
- 목표: 이동 시 "회수 먼저", 퇴사 시 "즉시 비활성화" 원칙.
- 핵심: [onboarding-transfer-offboarding.md](onboarding-transfer-offboarding.md).
- 실습: 퇴사 처리 체크리스트 따라 계정 비활성화.
- 주의: 마지막 관리자 강등·비활성화 불가 → 사전 지정.
- 확인 질문: 부서 이동 시 첫 번째 할 일은? (답: 불필요한 기존 권한 회수)

## 8. 사고 대응과 감사로그 (5분)
- 목표: 대표 사고 대응·장애 확인 순서 숙지.
- 핵심: [permission-incident-response.md](permission-incident-response.md), fail-closed 원칙.
- 실습: 감사로그에서 최근 권한 변경 조회.
- 주의: 권한 우회 임시코드·fail-open 금지.
- 확인 질문: "범위 밖 데이터가 보인다"의 심각도와 즉시 조치는?

---

## 교육 후 평가 (10문항 예시)
1. 기본 역할 4종은? 2. 하자접수 담당자가 볼 수 있는 메뉴는? 3. additive와 restrictive 차이? 4. 같은 유형 범위 결합 규칙? 5. 다른 유형 범위 결합 규칙? 6. 권한 변경 반영 시점? 7. 시스템 기본 권한 수정 가능 여부? 8. 퇴사 처리 1단계? 9. 마지막 관리자 처리 주의점? 10. 범위 밖 데이터 노출 시 대응?
