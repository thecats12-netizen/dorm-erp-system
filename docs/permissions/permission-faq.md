# 권한 운영 FAQ

**1. 메뉴가 안 보입니다.**
해당 메뉴 권한이 없거나, 선택(restrictive) 권한에서 그 메뉴를 선택하지 않았을 수 있습니다. 사용자관리 > 해당 계정 > 최종 권한 미리보기에서 "보이는 메뉴"를 확인하세요.

**2. 권한을 추가했는데 반영되지 않습니다.**
권한 변경은 **다음 로그인 또는 새로고침 후** 적용됩니다. 로그아웃 후 재로그인하세요.

**3. 다른 기숙사(지역) 데이터가 안 보입니다.**
데이터 범위가 특정 지역/기숙사로 제한된 것입니다. 미리보기의 "데이터 범위"를 확인하고, 필요하면 범위를 조정하세요.

**4. Excel(다운로드) 버튼이 없습니다.**
다운로드 권한(Excel/CSV/PDF)이 별도로 부여되지 않았습니다. 조회 권한과 다운로드 권한은 구분됩니다.

**5. restrictive와 additive의 차이는 무엇입니까?**
"기존 권한에 추가(additive)"는 기본 역할을 유지하며 권한을 더합니다. "선택한 메뉴만 허용(restrictive)"은 선택한 것만 쓰고 나머지는 숨깁니다(기본 역할 권한 제외).

**6. 여러 권한을 동시에 받을 수 있습니까?**
네. 여러 사용자 정의 권한을 배정할 수 있습니다. 선택(restrictive)이 하나라도 있으면 선택 우선, 모두 추가면 합집합입니다. 데이터 범위는 같은 유형 합집합·다른 유형 교집합입니다.

**7. 권한 삭제 후 복구할 수 있습니까?**
네. 삭제는 Soft Delete로 목록에서만 제외되며 복구 가능합니다. 복구 후 메뉴·기능·범위를 재확인하세요.

**8. 시스템 기본 권한을 수정할 수 있습니까?**
아니요. 관리자/뷰어/기숙사 관리자/하자접수 담당자는 수정·삭제할 수 없고 **복제만** 가능합니다.

**9. 퇴사자 계정은 어떻게 처리합니까?**
추가 권한 회수 → 계정 사용중지(비활성화) → 담당 데이터 인수인계 → 감사로그 보존. 완전 삭제는 보존 정책 확인 후에만.

**10. 관리자(admin) 권한은 몇 명까지 둘 수 있습니까?**
제한은 없으나 **최소 인원(권장 2~3명)** 을 유지합니다. 마지막 관리자 계정은 강등·비활성화가 차단됩니다.

**11. 하자접수 담당자에게 다른 메뉴를 줄 수 있습니까?**
아니요. 보호 정책으로 청소관리·하자접수만 보이며, 추가 권한 부여가 차단됩니다.

**12. 사용자에게 데이터 범위를 줬는데 목록이 0건입니다.**
선택(restrictive) 권한에서 범위를 지정하지 않으면 0건이 될 수 있습니다. 또는 조건이 실제 데이터와 맞지 않을 수 있습니다. 범위와 "전체 데이터" 여부를 확인하세요.

**13. "전체 데이터"를 선택했는데 지역도 선택되어 있습니다.**
전체 데이터를 선택하면 지역·성별·기숙사 등 조건이 자동 해제됩니다(전체가 우선). 조건별 제한을 원하면 전체를 해제하세요.

**14. 권한 코드는 왜 필요한가요? 화면에 보이나요?**
내부 식별용입니다. 영문 소문자·숫자·밑줄로 작성하며 중복·시스템 기본 코드는 불가합니다. 일반 사용자 화면에는 노출되지 않습니다.

**15. 권한을 복제하면 사용자 배정도 복제되나요?**
아니요. 메뉴·기능·적용 방식은 복제되지만 **사용자 배정은 복제되지 않습니다**. 데이터 범위·이름·코드는 재검토하세요.

**16. 권한 변경 이력은 어디서 봅니까?**
감사로그(권한 변경 기록)에서 변경자·변경 전/후·시각을 확인할 수 있습니다.

**17. 사용자 A로 로그인했는데 이전 사용자 권한이 남아 있나요?**
정상 구현상 로그아웃 시 권한 캐시가 초기화됩니다. 혹시 남아 있으면 즉시 보고(사고 대응 8번).

**18. viewer인데 등록·수정이 필요합니다.**
뷰어는 조회 전용입니다. 쓰기 업무가 필요하면 뷰어 대신 적절한 역할/추가 권한을 검토하세요.

**19. 임시(단기) 권한은 어떻게 관리하나요?**
데이터 범위에 적용 시작일/종료일을 지정하고, 월간 점검에서 만료분을 회수합니다.

**20. 다운로드 파일에 범위 밖 데이터가 포함될 수 있나요?**
정상 동작에서는 화면에 보이는 범위만 포함됩니다. 전체 데이터가 포함되면 **Critical 사고**로 즉시 대응하세요.

**21. 권한을 바꿨는데 사용자가 여전히 오류(403)를 봅니다.**
정상적인 권한 차단(403)일 수 있습니다. 접근하려는 메뉴/데이터가 실제 허용 범위인지 미리보기로 확인하세요. 허용인데 막히면 재로그인 후 감사로그·서버 로그를 확인하세요.

**22. 시험 공정 제한은 서버에서도 막히나요?**
네. 시험 공정 범위는 서버(RLS)에서 강제되어 URL/직접 호출로 우회할 수 없습니다.
