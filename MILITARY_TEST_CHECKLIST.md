## 🧪 테스트 절차

### Step 1: 인사 데이터 업로드
```
App.tsx 인사관리 탭 → 엑셀 업로드 → 군대관리_인사_샘플.xlsx 선택
```

**확인 사항:**
- [ ] 5명 모두 "예비군/민방위 인원 관리" 탭에 표시됨
- [ ] 각 인원의 이름, 연락처, 부서 확인
- [ ] 자동 계산된 "현재구분", "예비군연차", "민방위연차" 확인

| 이름 | 재직상태 | 현재구분 | 예비군연차 | 민방위연차 | 동원여부 | 필요훈련 | 상태 |

| 박성철 | 재직 | 예비군 | 6년차 | - | 동원미지정 | 기본훈련+작계훈련 14시간 | 재직 |

| 최미영 | 신규입사 | 대상아님 | - | - | 동원미지정 | (없음) | 신규입사 |

### Step 2: 자동 훈련기록 생성 확인

인사 추가 후, "훈련/교육 기록" 탭으로 이동하여 자동 생성된 레코드 확인.

#### 김준영 - 동원훈련 (1건)
```
대상자: 김준영
훈련유형: 동원훈련
차수: 1차
훈련일: (없음)
상태: 일정미등록
비고: 자동생성
```
- [ ] 1건 정확히 생성됨
- [ ] 상태: "일정미등록"
- [ ] 비고: "자동생성"

#### 이영숙 - 동미참훈련 (1건)
```
대상자: 이영숙
훈련유형: 동미참훈련
차수: 1차
상태: 일정미등록
비고: 자동생성
```
- [ ] 1건 정확히 생성됨
- [ ] **시간 확인: 32시간**

#### 박성철 - 기본훈련 + 작계훈련 (2건)
```
1) 대상자: 박성철, 훈련유형: 기본훈련, 차수: 1차, 상태: 일정미등록
2) 대상자: 박성철, 훈련유형: 작계훈련, 차수: 1차, 상태: 일정미등록
```
- [ ] 2건 정확히 생성됨
- [ ] 기본훈련: 8시간
- [ ] 작계훈련: 6시간

#### 이준호 - 민방위교육 (1건)
```
대상자: 이준호
훈련유형: 민방위교육
차수: 1차
상태: 일정미등록
비고: 자동생성
```
- [ ] 1건 정확히 생성됨
- [ ] 시간: 4시간

#### 최미영 - 없음 (0건)
```
훈련 기록 없음 (대상아님이므로 자동생성 불가)
```
- [ ] 자동 생성된 레코드 없음 ✓

---

### Step 3: 훈련 시간 상세 검증

#### 동원 vs 동원미지정 시간 구분
- [ ] 김준영 (동원): **28시간** ✓
- [ ] 이영숙 (동원미지정): **32시간** ✓
- [ ] 차이점 명확히 확인

#### 5~6년차 기본훈련 + 작계훈련
- [ ] 박성철: 기본훈련 **8시간** + 작계훈련 **6시간** = 총 14시간 ✓

#### 민방위 교육
- [ ] 이준호: 민방위교육 **4시간** ✓

---

### Step 4: 중복 생성 방지 테스트

#### 절차
1. 김준영 기존 레코드 확인
2. 김준영 수정 (예: 비고 변경) 후 저장
3. 동일한 훈련기록이 또 생성되지 않는지 확인

#### 기대값
- [ ] 훈련기록이 중복 생성되지 않음
- [ ] 기존 레코드 개수 유지 (김준영은 1건)

---

### Step 5: 상태 변경 시 "관리제외" 처리 확인

#### 절차
1. 김준영 재직상태를 **"퇴사"**로 변경 후 저장
2. "훈련/교육 기록" 탭 재확인

#### 기대값
```
대상자: 김준영
훈련유형: 동원훈련
상태: 관리제외 (← 변경됨!)
```
- [ ] 상태가 "관리제외"로 변경됨
- [ ] 새로운 훈련기록이 추가로 생성되지 않음
- [ ] 감사 로그에 업데이트 기록 남음

#### 복구 (선택사항)
1. 김준영 재직상태를 다시 **"재직"**으로 변경
2. 상태가 어떻게 되는지 확인

---

### Step 6: 대시보드 집계 확인

"예비군/민방위 인원 관리" 탭의 대시보드 통계:

```
전체 인원: 5명
├─ 자동생성 대상 인원: 4명 (김준영, 이영숙, 박성철, 이준호)
└─ 대상아님: 1명 (최미영)

훈련기록 현황:
├─ 총 생성된 레코드: 5건
│  ├─ 김준영: 1건 (동원훈련 28시간)
│  ├─ 이영숙: 1건 (동미참훈련 32시간)
│  ├─ 박성철: 2건 (기본훈련 8시간 + 작계훈련 6시간)
│  └─ 이준호: 1건 (민방위교육 4시간)
└─ 최미영: 0건 (대상아님)
```

- [ ] 각 셀의 "필요훈련" 항목 확인
- [ ] 각 셀의 "필요시간" 항목 확인
- [ ] 각 셀의 "이수상태" 항목 확인 (미이수 상태)
- [ ] 각 셀의 "훈련기록수" 수 정확함

## 🐛 문제 발생 시

### Q: 자동 훈련기록이 생성되지 않는다
A: 다음을 확인하세요:
1. [ ] 재직상태가 **"재직"**인가? (기본 설정)
2. [ ] 군대관리 설정 탭에서 자동생성 ON 상태?
3. [ ] 브라우저 개발자 도구 콘솔 에러 확인
4. [ ] LocalStorage `MILITARY_TRAINING_AUTOCREATE_KEY` 확인

### Q: 훈련 시간이 잘못 표시된다
A: 다음을 확인하세요:
1. [ ] 동원여부 값 정확한가? ("동원" vs "동원미지정")
2. [ ] 연차 계산 정확한가? (전역일 기준)
3. [ ] `defaultMilitaryTrainingRules` 규칙 확인

### Q: 중복 생성되거나 누락된 레코드가 있다
A: 다음을 확인하세요:
1. [ ] 브라우저 개발자 도구 Network 탭 확인
2. [ ] LocalStorage 수동 확인: `MILITARY_TRAINING_KEY`
3. [ ] 감사 로그 확인: 중복 생성 기록 있는가?

## 📝 추가 테스트 시나리오 (선택사항)

### 시나리오 1: 자동생성 설정 OFF 상태
1. 군대관리 설정 → 자동생성 OFF로 변경
2. 새 인사 추가
3. 훈련기록이 생성되지 않는지 확인
4. 기대값: 자동 생성되지 않음 ✓

1. 군대관리 설정 → 기준연도를 2025로 변경
2. 인사 목록 새로고침
3. 연차 계산이 변경되는지 확인
4. 필요훈련이 변경되는지 확인

### 시나리오 3: 수동 모드 병역구분
1. 최미영 계산모드를 "manual"로 변경
2. 병역구분을 "예비군"으로 수동 설정
3. 필요훈련이 "동원훈련"으로 변경되는지 확인
4. 자동 훈련기록이 생성되는지 확인
